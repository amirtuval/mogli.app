# Tabnine Project Context: Mogly

## What is this project?

Mogly is a cross-platform desktop app (Tauri v2 + React/TypeScript) providing a unified inbox and calendar for multiple Google accounts. Currently in **Phase 1: Scaffold + OAuth + Debug View**.

The full design spec lives at `docs/mogly-design.md`.

## Tech stack

- **Frontend:** React 19, TypeScript (strict), Vite 7, Zustand (UI state), TanStack Query (server state)
- **Backend:** Rust 2024 edition (Tauri v2), reqwest (rustls TLS), keyring (OS keychain), serde
- **Type bridge:** tauri-specta → auto-generates `src/types/bindings.ts` from Rust command signatures
- **Styling:** CSS Modules with CSS custom properties for theming (no Tailwind)
- **Testing:** Vitest + React Testing Library (frontend), cargo test + mockito (Rust)
- **Package manager:** Yarn Classic (v1.22.x) — not npm

## Project layout

```
src/                      # React/TypeScript frontend
  store/uiStore.ts        # Zustand store
  types/bindings.ts       # Auto-generated by tauri-specta (do not edit)
  hooks/                  # React custom hooks
  __tests__/              # Frontend tests (Vitest)
  App.tsx                 # Root component
  main.tsx                # Entry point

src-tauri/src/            # Rust backend
  main.rs                 # Entry point → calls mogly_lib::run()
  lib.rs                  # Tauri builder, plugin registration, specta setup
  models.rs               # Shared data structs (Account, MessageMeta, Thread, etc.)
  store.rs                # AccountStore — JSON persistence via tauri-plugin-store
  keychain.rs             # OAuth token storage via OS keychain
  commands/               # Tauri commands exposed to frontend
    auth.rs               # add_account, remove_account, list_accounts
    gmail.rs              # get_messages, get_thread
  google/                 # Google API clients
    oauth.rs              # OAuth2 loopback flow, token refresh
    gmail.rs              # Gmail REST API client
```

## Build & run commands

```bash
yarn tauri dev              # Dev mode (frontend + Rust backend)
yarn tauri build            # Production build
yarn test                   # Frontend tests (Vitest, non-watch)
yarn lint                   # ESLint
yarn format:check           # Prettier check
cargo test --manifest-path src-tauri/Cargo.toml --all   # Rust tests
cargo fmt --check --manifest-path src-tauri/Cargo.toml  # Rust format check
cargo clippy --manifest-path src-tauri/Cargo.toml -- -W clippy::pedantic -D warnings  # Rust lint
npx tsc --noEmit            # TypeScript typecheck
```

## Code style rules

### Rust
- Clippy **pedantic** is enabled; warnings are errors (`-D warnings -W clippy::pedantic`)
- Specific pedantic lints are allowed in `lib.rs`: `missing_errors_doc`, `missing_docs_in_private_items`, `module_name_repetitions`
- `rustfmt.toml`: edition 2024, max_width 100, use_field_init_shorthand
- Tauri commands return `Result<T, String>` with human-readable error strings
- Do NOT use `#[allow(dead_code)]` — remove unused code or comment it out with a note about which phase will add it back

### TypeScript / React
- Strict mode (`noUnusedLocals`, `noUnusedParameters`)
- Prettier: no semicolons, single quotes, trailing commas, 100 char width
- ESLint with react-hooks and react-refresh plugins
- Functional components with hooks only

## Debug vs release isolation

Dev and production builds use completely separate storage so the app can be developed and used as a daily driver on the same machine:

| Resource        | Debug (`yarn tauri dev`)    | Release (`yarn tauri build`) |
| --------------- | --------------------------- | ---------------------------- |
| Keychain service | `mogly-dev`                | `mogly`                      |
| Store filename  | `accounts.dev.json`         | `accounts.json`              |
| Window title    | `[DEV] Mogly`               | `Mogly`                      |

Switching is automatic via `cfg(debug_assertions)` — no env vars or flags needed.

## Key conventions

- Frontend ↔ backend communication is exclusively via Tauri commands (never direct HTTP from frontend)
- OAuth tokens are stored in the OS keychain, never in files or frontend state
- Account config is persisted as JSON via `tauri-plugin-store` in the app data directory
- No UI element is built until the feature behind it is implemented — no placeholder buttons
- Environment variables (`GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET`) are loaded from `.env` at runtime; never committed

## CI pipeline

GitHub Actions (`.github/workflows/ci.yml`) runs on every push/PR:
1. `cargo fmt --check` + `cargo clippy -- -W clippy::pedantic -D warnings`
2. `yarn format:check` + `yarn lint`
3. `cargo test --all`
4. `yarn test`
5. `npx tsc --noEmit`
